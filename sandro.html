Introdução a protocolos

É uma tecnologia que permite a comunicação entre os computadores, se tiver duas pessoas que não falam a mesma lingua, não tem comunicação.

Função do protocolo

É a mesma do tradutor, ou seja possibilita a comunicação entre os computadores

Variados protocolos

ARP,HTTP,FTTP,TCP,ICMP,IP,SMTP,TELNET,UDP e NNTP

Funções dos protocolos

ARP descobrir o endereço MAC, da máquima para qual determinado pacote IP deve ser entregue.

HTTP permite a obtenção de recursos, como documentos HTML, base na troca de dados na WEB.

FTP permite a troca de arquivos entre computadores conectados a internet.

TCP garantir que o destinário receba todos os pacote, ou seja um PC ou um servidor.

ICMP não está associado a nenhum protocolo de camada de transporte como TCP ou UDP.

IP é o identificador que permite que as informações sejam enviadas entre dispositivos de uma rede.

SMTP responsável por garantir que os e mails que você envia diário chegue aos destinátarios.

TELNET usado para agilizar o conserto de falhas nos computadores.

UDP funciona igual ao TCP, mas elimina todo o material de verificação de erros.

NNTP protocolo da internet para grupos de discussão da chamada usenet.

Funções dos protocolos - TÓPICOS

SINTAXE É o padrão dos dados e a forma sequenciada que os dados são mostrados.

SEMÂNTICA Representa um padrão de dados, para dar um significado à mensagem.

TIMING Estabelece uma velocidade de transmissão dos pacotes(pedaços da mensagem).

PROTOCOLO TEM FUNÇÕES PREDETERMINADA PARA OBTER COMUNICAÇÃO ENTRE HOSTS

Endereçamento determina qual o destino da mensagem para enviar uma carta necessita de um destinátario.

Sequencia e numeração identifica as mensagens por meio de numero sequencial.

Estabelecer a conexão constitui um canal lógico entre duas pontas.

Controle de erros idêntica e corrigi os erros durante a comunicação.

Retransmissão Se o sinal ACK não é recebido no destinário, ou quando a mensagem não é recebida no destino.

Confirmação de recebimento Envia uma confirmação para cada pacote recebido ACK.

Conversão de código Faz um ajuste dos códigos de acordo ao destinatário.

CADA PROTOCOLO É DEFINIDO UM PADRÃO, EXISTE 2 TIPOS DE PADRÃO:

Facto padrões usados pela comunidade, por fabricantes quando lançam novos produtos.

Jure usados pelos fabricantes quando lançam novos produtos, sem a aprovação dos comitês reconhecidos.
